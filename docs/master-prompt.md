# LegalPath Master Prompt

## Objective
Finalize the visual and functional polish of the LegalPath platform, ensuring a seamless experience for both users and lawyers while adhering to the established branding and security standards.

## Branding Guidelines
- **Project Identity:** LegalPath - Chilean legal marketplace.
- **User Branding (Mint):** Hex: #1ECCA7. Used for general users seeking legal help.
- **Lawyer Branding (Orange):** Hex: #EE6C4D. Used for professional lawyers seeking clients.
- **Typography:**
  - Headlines: "Plus Jakarta Sans"
  - Body: "Inter"

## Architecture (A.N.T. Protocol)
- **Layer 1: Architecture (`architecture/`)** - SOPs and Technical Specifications.
- **Layer 2: Navigation** - Agentic reasoning and tool routing.
- **Layer 3: Tools (`tools/`)** - Deterministic execution scripts.

## Core Workflows
1. **Case Submission:** 3-step funnel (Description → Auth → Success).
2. **Lawyer Onboarding:** Multi-block registration (Sign up → Profile → Verification).
3. **Dual Landing:** Home Page (Users) & Abogados Page (Lawyers).

## Principles
- **Wow Factor:** Premium design, smooth animations, and high-quality UI assets.
- **Anonymity:** Absolute privacy for users until the final "Match".
- **Performance:** Fast loading times and responsive layouts.
